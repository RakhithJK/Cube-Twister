The Notation Editor - Reflection

An reflection is used to specify a reflection of a statement.

A reflected face turn is applied to the opposite face of the cube and in the opposite turn direction.

A reflection has no effect on mid-layer twists, cube turns and permutations.

Please note: Reflections are available only, if quarter turn face twists are enabled.

The «Reflection» checkbox:

Tick this checkbox to enable support for reflections by the notation.

When this checkbox is ticked, then additional fields are shown below the checkbox.

«Reflector» text fields:

Use this text field to specify the keyword that denote the reflector.

You must fill in this field, if reflections are enabled.

If you enter multiple words in the text field, then each word represents a keyword. The first word in the text field is used by CubeTwister for output of scripts in this notation.

«Position» radio buttons:

Use these radio buttons to specify the location of the reflector

Example 1:

If you fill in the text fields as shown in the table below, then the statement «A» is reflected by specifying «reflect A».

Reflector:  reflect 
Position:  leading

Example 2:

If you fill in the text fields as shown in the table below, then the statement «A» is inverted by specifying «A*».
